DGI vs DCII: Framework vs Implementation

Understanding the relationship between the open standard (DGI) and Datacendia's reference implementation (DCII)

OPEN STANDARD
DGI
Decision Governance Infrastructure — A vendor-neutral framework for institutional decision accountability
Purpose: Define conceptual governance primitives independent of any software architecture

Primitives: 5 core (A–E)
Scope: All institutional decisions
License: Open framework with attribution
Audience: Standards bodies, governance architects, any organization
REFERENCE IMPLEMENTATION
DCII
Decision Crisis Immunization Infrastructure — Production-ready implementation with AI governance extensions
Purpose: Operationalize DGI with domain-specific extensions for AI governance and crisis prevention

Primitives: 9 core + 1 extended (P1–P10)
Scope: AI-assisted decisions in regulated industries
License: Commercial platform
Audience: Enterprises requiring production deployment

Key Differences

Scope
DGI: All institutional decisions across any domain

DCII: AI-assisted decisions in regulated industries (finance, healthcare, legal, government)
Purpose
DGI: Define conceptual governance primitives for standards adoption

DCII: Prevent institutional crises through cryptographic evidence infrastructure
Deployment
DGI: Framework only — organizations implement using their own architecture

DCII: Production SaaS, private cloud, on-premises, or air-gapped deployment
Extensibility
DGI: Explicitly allows domain-specific primitive extensions

DCII: Extends DGI with P6 (Bias), P9 (Jurisdiction), P10 (Outcome)
Measurement
DGI: DGMI (5-level maturity index) — qualitative governance assessment

DCII: IISS (0–1000 score) — quantitative resilience measurement
Target Audience
DGI: Standards bodies, governance architects, policy makers

DCII: CIOs, CROs, compliance officers, legal teams in regulated enterprises
